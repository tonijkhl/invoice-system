Invoice Generator API

Endpoint

The primary endpoint for generating invoices is:

http://your-api.com/generate_invoice

Required Headers

X-API-KEY

Your unique API key for authentication.

Content-Type

Must be set to application/json.

Request Example (cURL)

Here's an example of a cURL request with a JSON payload:

curl -X POST http://your-api.com/generate_invoice \ -H 'X-API-KEY: YOUR_API_KEY' \ -H 'Content-Type: application/json' \ -d '{ "client_name": "Acme Corp", "client_email": "contact@acmecorp.com", "items": [ { "description": "Consulting Services", "quantity": 2, "unit_price": 150.00 }, { "description": "Project Management", "quantity": 1, "unit_price": 200.00 } ], "tax_rate": 0.05 }'

JSON Payload Fields

client_name

(String) The name of the client.

client_email

(String) The email address of the client.

items

(Array) An array of items in the invoice.

description

(String) Description of the item.

quantity

(Integer) Quantity of the item.

unit_price

(Float) Price per unit of the item.

tax_rate

(Float) Tax rate as a decimal (e.g., 0.05 for 5%).

Optional Parameters

logo

(File) Upload a logo image file to be included in the invoice.

Endpoint Functionality

This endpoint generates a PDF invoice based on the provided data. Upon successful request, it returns a PDF file. The file can be downloaded or displayed directly in the browser.

Expected Responses

200 OK

Invoice generated successfully. Returns the PDF file.

400 Bad Request

Invalid request data. Check the JSON payload.

401 Unauthorized

Invalid API key.

500 Internal Server Error

An unexpected error occurred on the server.

Handling the PDF Response

The response will include the PDF file as binary data. You can handle this in your application by saving the file or displaying it to the user. Ensure your application correctly handles binary data and sets the appropriate content type for the response (application/pdf).

Error Handling

In case of errors, the API will return a JSON response with an error message. For example:

{ "error": "Invalid client email format" }
